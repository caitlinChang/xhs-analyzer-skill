# 小红书博主分析 Skill

## 触发方式
`/xhs <博主昵称>` 或 `/xhs <主页链接>`

示例：
- `/xhs 栗噔噔`
- `/xhs https://www.xiaohongshu.com/user/profile/550aad1aa46e96256f711e78`

---

## 执行步骤

### Step 0：解析参数

从用户输入中提取目标博主信息：
- 如果包含 `/user/profile/`，提取后面的 `user_id`，使用 `--user-id` 参数
- 否则视为昵称，使用 `--name` 参数
- 如果用户指定了抓取数量（如「抓100篇」「最新50篇」），记录为 `limit`；否则默认 `limit=200`
- `limit=0` 表示抓取全部

### Step 1：检查 Cookie 配置

读取 `~/.claude/xhs_config.json`：

```json
{
  "cookies": "a1=xxx; web_session=xxx; ..."
}
```

**如果文件不存在或 cookies 为空**，提示用户按以下步骤获取：

> 需要你的小红书登录凭证，请按以下步骤操作：
> 1. 用 Chrome 打开 https://www.xiaohongshu.com 并登录
> 2. 按 F12 → Application（应用程序）→ Cookies → www.xiaohongshu.com
> 3. 将所有 cookie 拼成 `key=value; key=value` 格式粘贴给我

收到后写入配置文件：
```json
{ "cookies": "<用户粘贴的内容>", "saved_at": "<当前时间>" }
```

**如果爬取中途发现 Cookie 失效**（脚本退出码为 2），提示用户重新提供并更新配置后重跑。

### Step 2：运行爬虫脚本

使用 Bash 工具执行（根据输入类型选择参数）：

```bash
# 按 user_id（默认抓取200篇）
python3 ~/.claude/scripts/xhs_scrape.py --user-id <USER_ID> --cookies-file ~/.claude/xhs_config.json --limit <LIMIT>

# 按昵称（默认抓取200篇）
python3 ~/.claude/scripts/xhs_scrape.py --name "<昵称>" --cookies-file ~/.claude/xhs_config.json --limit <LIMIT>
```

`<LIMIT>` 默认填 `200`；用户指定数量时填用户说的值；用户要求抓全部时填 `0`。

脚本输出文件到 `/tmp/xhs_raw/`：
- `profile.json` — 博主基础信息
- `notes.json` — 笔记列表（含标题、类型、点赞、正文、发布时间）

脚本预计耗时：笔记数量 × 2 秒，运行时实时输出进度。

### Step 3：读取原始数据

用 Read 工具读取：
- `/tmp/xhs_raw/profile.json`
- `/tmp/xhs_raw/notes.json`

### Step 4：Claude 分析

基于数据完成以下 7 个维度的分析：

#### 4.1 基础信息
整理：昵称、简介、IP属地、粉丝数、关注数、获赞收藏总数、赞藏/粉丝比（高于 5x 为高互动）

#### 4.2 内容数据总览
- 笔记总数、视频/图文占比
- 累计点赞、平均点赞/篇
- Top 10 爆款笔记表格（排名、类型、标题、点赞、发布时间、内容摘要）

#### 4.3 内容主题聚类
阅读所有笔记标题和正文，归纳主题分类（根据内容实际情况灵活命名，通常 4~7 个类别）。
每个分类输出：笔记数量、占比、平均点赞、代表笔记 3 篇。

#### 4.4 发布时间规律
利用笔记 ID 解析发布时间（高位 4 字节为 Unix 时间戳）：
```python
ts = int(note_id[:8], 16)
dt = datetime.fromtimestamp(ts)
```
分析：
- 发布时间跨度（最早 → 最新）
- 平均更新频率
- 高产期 vs 沉寂期

#### 4.5 账号转型轨迹
按时间顺序分析内容方向演变：
- 早期内容方向
- 转折节点（何时、为何）
- 当前聚焦方向
- 转型效果（互动数据对比）

#### 4.6 爆款内容归因
分析点赞 Top 20% 笔记的共同特征：
- 标题格式（数字型/疑问型/教程型/情绪型等）
- 内容类型（教程 vs 观点 vs 故事 vs 资讯）
- 话题标签规律
- 结论：该账号最容易爆的内容公式

#### 4.7 竞品洞察建议
站在「我要做对标账号」视角：
- 该博主核心竞争力（可复制的部分）
- 内容空白点（受众有需求但博主未覆盖）
- 值得借鉴的选题公式
- 需要差异化的地方

### Step 5：询问输出方式

分析完成后询问：

> 分析完成！报告输出到哪里？
> **1. 飞书文档**（默认，回车即选）
> **2. 本地 MD 文件**（输出到当前目录）
>
> 请输入 1 或 2：

### Step 6：生成两份文档

#### 文档一：账号分析报告

文件名：`{昵称}_账号分析报告_{YYYYMMDD}.md`

```markdown
# {昵称} 小红书账号分析报告

> 分析日期：{日期} | 笔记总数：{N} 篇 | 数据来源：小红书公开主页

## 一、基础信息
## 二、内容数据总览
## 三、Top 10 爆款笔记
## 四、内容主题分布
## 五、发布时间规律
## 六、账号转型轨迹
## 七、爆款内容归因
## 八、竞品洞察建议
```

#### 文档二：全量笔记列表

文件名：`{昵称}_全量笔记_{YYYYMMDD}.md`

按点赞降序，每篇包含：排名、类型、标题、点赞数、发布时间、内容摘要。

#### 输出到飞书

用 `mcp__lark__docx_builtin_import` 分别导入两份文档（`useUAT: true`），然后将返回的链接告知用户，并提示将文档移入目标知识库。

#### 输出到本地

用 Write 工具写入当前工作目录。

## 错误处理

| 情况 | 处理 |
|------|------|
| Cookie 失效（脚本退出码 2） | 提示重新提供 Cookie，更新配置后重试 |
| 找不到博主 | 提示确认昵称，或直接提供主页链接 |
| 笔记正文获取失败 | 记为「(未获取到正文)」，不中断流程 |
| Playwright 未安装 | 提示运行 `python3 -m playwright install chromium` |
| 脚本不存在 | 提示重新执行安装脚本 `install.sh` |
